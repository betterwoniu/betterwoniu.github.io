原型链

每个对象都有一个原型 _proto_ ，这个原型对象还可以有自己的原型对象，以此类推形成一个原型链。查找特定的属性先在这个对象属属性中寻找，如果没有就去它的原型对象中查找，如果没有的话就从这个对象的原型对象的原型对象中查找；这个操作就被委托在原型链中，这个就是原型链。

  1. 每个对象的都有一个_proto_ 属性，并且指向它的prototype 原型对象
  2. 每个构造函数都有一个prototype 原型对象
  3. prototype 原型对象里的constructor函数
    实例函数的 _proto 指向构造函数的prototype,从而实现原型链继承

rxjs

Rxjs note

Rxjs 是一个库，它通过使用observable序列来编写异步和基于事件的程序。它提供了一个核心类型Observable,附属类型（ Obsever、Schedules、Subjects）和受[Array#extras]启发的操作符（map、filter、reduce、every）,这些数组操作符可以把异步事件作为集合来处理。结合了观察者模式、迭代器模式和使用集合的函数式编程，以满足以一种理想方式来管理事件序列所需要的一切。

在Rxjs用来解决异步事件管理的基本概念：

1.Observable(可观察对象)：表示一个概念，这个概念是一个可调用的未来值或事件的集合。

Observable 是使用Rx.Observable.create或创建操作符创建的,并使用观察者（observer）观察它.然后它发送next/error/complete 通知观察者。

创建Observable的方式：

  1. create(subscribe: function)，使用给定的订阅函数创建
  2. empty(scheduler: Scheduler): Observable, 创建立即完成的 observable
  3. from(ish: ObservableInput, mapFn: function, thisArg: any, scheduler: Scheduler): Observable, 将数组、promise 或迭代器转换成 observable
  4. fromEvent(target: EventTargetLike, eventName: string, selector: function): Observable, 将事件转换成 observable 序列
  5. fromPromise(promise: Promise, scheduler: Scheduler): Observable, 创建由 promise 转换而来的 observable，并发出 promise 的结果。
  6. interval(period: number, scheduler: Scheduler): Observable, 基于给定时间间隔发出数字序列
  7. of(…values, scheduler: Scheduler): Observable 按顺序发出任意数量的值
  8. range(start: number, count: number, scheduler: Scheduler): Observable 依次发出给定区间内的数字。
  9. throw(error: any, scheduler: Scheduler): Observable 在订阅上发出错误
  10. timer(initialDelay: number | Date, period: number, scheduler: Scheduler): Observable 给定持续时间后，再按照指定间隔时间依次发出数字

2.Observer(观察者)：一个回调函数的集合，它知道如何去监听由Observable提供的值

3.Subscription(订阅)：表示Observable的执行，主要用于取消Observable的执行

4.Subject(主体)：相当于EventEmitter，并且是将值或事件多路推送给多个Observer的唯一方式

5.Scheduler(调度器)：用来控制并发并且是中央集权的调度员，允许我们在发生计算时进行协调，例如 setTimeout或 requestAnimationFrame 或其他

缓存

缓存之cookies、session、sessionStorage和localStorage

cookies 和 session

  1. cookies 是存储在客户端，session是存储在服务器端

  2. cookies机制:如果不再浏览器中设置cookies的过期时间，cookies将会保存在内存中，生命周期随着浏览器关闭而结束，这种cookie称之为会话cookies，如果给cookies设置了过期时间，则是将cookies存储在硬盘中，随着过期时间结束才消失。cookies是服务器发送给客户端的特殊信息，cookies是以文本的形式存储在客户端，当每次发送请求的时候都会带上cookies

  3. cookie主要用户存储浏览器数据

  4. cookie以文本的形式存储

  5. 每一条cookie大小为4k左右

  6. cookie会随着http 发送到服务端

  7. 在同一域名下是可以共享的

  8. cookie 的name 为cookie的名字，value 为cookie的值 domain 为cookie的作用域名

  9. 过期时间：expires 和max-age 的作用相同设置过期时间，但是max-age的作用级优先expires max-age的值为秒，expires值为过期时间，

  10. httponly：控制是否允许脚本获取cookie

  11. session机制：当服务端收到请求需要创建session对象时，首先会检查请求中是否包含sessionId，如果包含sessionId，则服务端会根据sessionId找到对应的session对象，如果没有sessionID，服务端则会创建新的session对象，并把sessionId返回给客户端。通常使用cookies的形式将sessionId存储到客户端。

  12. cookies是以字符串的形式，将信息以文本的形式存储到硬盘，session是以hashtable的数据结构形式保存，能支持任何类型的对象（session可能包含多个对象）

  13. cookie是有大小限制的，单个cookies的大小不能超过4kb，session的大小是没有限制的

sessionStorage和localStorage——html5的本地储存方式

针对H5对于cookie存储机制的一个改善，H5转而使用改良后的Web Storage存储机制
web storage 分为sessionStorage和localStorage：

sessionStorage机制：

  1. sessionStorage在浏览器窗口打开期间将会一直存在，重新加载或恢复页面将会仍然保持
  2. 在页面中打开一个新的页面的时候将会复制父级页面的sessionstorage
  3. 多个相同url的页面的sessionstorage 是相互隔离的
  4. 关闭浏览器窗口的时候将会清除当前页面的sessionstorage

    localstorage 机制：

  5. localstorage 会长期保留，当页面关闭之后数据仍然存储在本地存储中
  6. 允许在同一个域名下的各个页面相互访问
  7. sessionStorage机制：将数据保存到session对象中，用于临时保存同一窗口(或标签页)的数据，在关闭窗口或标签页之后将会删除这些数据（临时保存）
  8. 将数据保存在客户端本地的硬件设备(通常指硬盘，也可以是其他硬件设备)中，即使浏览器被关闭了，该数据仍然存在，下次打开浏览器访问网站时仍然可以继续使用（永久保存）
  9. sessionStorage和localStorage的存储大小一般都是5MB，不与服务器相互通信，只能存储字符串类型

javascript

执行上下文

  • 定义：当js引擎执行代码片段的时候就会先进行执行前的准备工作，这个准备工作就叫做执行上下文。所有执行上下文都会交给“执行上下文堆”，全局上下文会在该栈的最底部，每一个函数上下文执行的时候，就会将函数的执行上下文压入栈。但是栈的容量是有限制的，当容量超过限制的时候就会造成栈溢出。
  • 执行上下文中包含了：声明的变量（变量声明会触发变量和函数提升）、this指向、作用域链
  • 函数的内部执行会先访问本地变量，如果找不到就会通过作用域链去寻找上层变量
  • this的指向取决于谁去调用它，apply、bind、call函数是可以改变this的指向

作用域/作用域链

  • 作用域定义：当前程序定义变量的区域，它决定了执行当前代码对变量访问的权限（分为全局作用域和函数作用域）
  • 作用域链定义：代码执行的之后会在本地寻找变量，如果有的话就返回，如果没有的话就向上级作用域去寻找，一直找到全局作用域

闭包

  • 父函数的内部返回一个内部子函数，内部子函数使用了父函数内的变量，父函数外部持有内部子函数的引用，即形成了闭包
  • 闭包导致的内存泄漏问题：js的垃圾回收机制是，每一个变量被引用的时候就会在该变量的引用次数加1，当一个变量的引用次数为0的时候就会清除掉该变量，但是，在一个函数中使用另外一个函数的时候，该函数的引用次数始终为1，而在另外的函数中可能也会使用其他地方的变量，那么涉及到其他的变量就无法被销毁始终存在栈中，闭包使用过度就会造成内存无法被释放，这个就是闭包造成的内存泄漏
  • 内存泄漏方案：1.避免使用全局变量；2.谨慎使用dom绑定事件；3、尽量少使用闭包
  • 例如下面的节流和防抖就是在使用闭包：DOM元素触发的事件持有内部函数的引用，而内部函数中使用定义的timer定时器返回的timer变量

this 指向问题

  • this的指向一般指的的是 当前作用域的触发的函数，由谁去触发这个函数，这个this就指向谁
  • apply 、call和bind函数除外，他们的this指向是传入的第一个参数

原型/原型链

深浅拷贝

  • 浅拷贝定义：对于复杂的引用类型的数据(对象或者数组)拷贝来说，浅拷贝只是拷贝的这个值的内存地址，内存地址指向的值改变会影响到所有使用该内存地址的数据。

  • 深拷贝：对于复杂的引用类型的数据(对象或者数组)拷贝来说，深拷贝是将内存中存取的数据拷贝一份，存放到一个新的内存地址，修改内存中的数据不会相互影响。

  • 深拷贝解析：先根据源数据的类型进行判断，如果是基本数据类型直接返回；如果是正则或者日期数据类型，使用 new Date()或者new Regex()新创建一个；如果是对象或者数据，需要对数据进行递归循环，为了防止对象的属性间接或直接的引用了自身的情况，所以需要新创建一个空间来存储新拷贝的对象(新创建的数据类型满足的数据类型：1.key-value形式；2:因为数据的引用自身，所以key需要是引用类型。因为使用map是抢数据类型，不能被垃圾回收器回收(置空key后，map仍然对于引用数据类型的key存在强引用数据类型)，所以需要使用弱数据类型WeakMap)

    事件循环机制

  • javascript中的任务分为同步和异步，同步任务是放在主线程中依次执行，异步任务存放在调用任务队列中排队等待，异步任务下一步会存放在调用栈中等待主线程执行。因为javascript是单线程的，所以每次只做一件事，主线程在执行异步任务等待响应的时候就会去执行其他任务，浏览器将回调函数注册到事件表中，等待响应结果到达之后，将回调函数添加到任务队列中等待执行

  • 宏任务：整体代码、settimeout、setInterval、IO、setImmediate、UI render

  • 微任务：promise、process.next()、async/await、MutationObserver

  • 宏任务和微任务执行过程：

  1. 完成某一个宏任务之后，检查微任务队列中是否有微任务
  2. 1）如果有微任务，则开始执行微任务，直到微任务队列中没有任何微任务了
    2）开始执行宏任务队列中最靠前的宏任务，
    3 如果执行宏任务的过程中，遇到微任务，则先讲微任务存放到微任务队列中，等待宏任务执行完以后，开始执行微任务队列
  3. 如果没有微任务，则继续执行宏任务

javascript 观察者模式和发布者模式

观察者模式

观察者模式是观察者监听着所观察的对象，当所观察的对象发生变化的时候，所有依赖于它的对象将会收到通知并进行相应的操作。观察者和所观察的对象是多对一的关系，一个被监听的对象对应着多个监听的对象。观察者和所观察的对象之间是相互交互的。

发布者模式

发布者模式是由调度中心监听发布者的变化，然后对依赖于它的所有订阅者进行相应的操作。发布者和订阅者之间是通过调度中心来交互的，一方面是解耦操作，另一方面是通过调度中心来进行更加细粒度的控制。

Object.defineProperty 和 proxy

Object.defineProperty

Object.defineProperty(obj, prop, descriptor) 参数：obj 要定义的属性的对象；prop：要修改的属性的名称；descriptor：要定义或修改的属性描述符。

对象目前存在的属性描述符有两种主要形式：数据描述符和存取描述符。数据描述符是一个具有值的属性，该值是可写的也可以是不可写的。存取描述符是由getter函数和setter函数所描述的函数的属性。一个描述符只能是两者之一；不能同事用两者。这两种描述符都是对象，它们共享以下可选键值

configurable：当且仅当该属性的 configurable 的值为true时，该属性的描述符才能被改变，同时该对象的属性也是可以被删除的（默认false);
enumerable: 当且仅当该属性的值为true时,该属性才会出现在对象的枚举属性(默认false);

数据描述符可选键值：

value:该属性对应的值。可以是任何js的有效值(数值，对象，函数等)(默认undefind)。

writable:当且仅当该属性的waitable属性为ture时，则该属性的值(上面所述的value)才可被赋值运算符所改变。

数据存取符可选键：
get:属性的getter函数，如果没有getter函数，则为undefind。当访问属性的值时，会调用该函数。执行时不传入任何参数，但是会传入this对象，该函数的返回值会被用作属性的值。
set:属性的setter函数，当属性值被修改时，会调用此函数。该方法接收一个参数(被赋值的新值)，会传入赋值时的this对象。

grid

grid 网格布局

  • grid 网格布局是将页面分为行和列，产生单元格，指定 元素 到指定的单元格
  • 页面采用网格布局的页面叫做容器，容器内部采用网格定位的子元素叫做 项目(item)

容器元素

  • 通过设置 display:grid 来让容器元素采用网格布局
  • 默认情况下 容器元素默认的是块状元素，通过设置display:inline-grid 来控制容器元素采用网格布局，并设置容器元素为行内元素
  • 容器元素设为网格布局之后，子元素(item)的float、display：inline-bloack和display：table-cell、vertical-align和column-*都失效

容器子元素的宽高

  • 通过设置grid-template-columns 和 grid-template-rows来控制item元素的 宽高
  • grid-template-columns 和 grid-template-rows的值为100px 100px 100px 或者 33.33% 33.33% 33.33% 采用百分比

flex 布局

flex 父元素属性

  1. flex-direction: 控制dom元素的排列方向，值为row，row-reverse,column，column-reverse
  2. justify-content: 控制子元素的在父元素中的水平排列方式
  3. align-content: 多根轴线下 垂直排列方式
  4. align-items: 单根轴线 垂直排列方式
  5. flex-wrap:控制子元素排列不下之后的排列方式， nowrap 一行排列，wrap 下面一行再排列 wrap-reverse 第二行在上面，第一行在下面
  6. flex-flow: flex-direction和flex-wrap 的结合

flex 子元素的属性

  1. order:排列的顺序
  2. flex-grow:剩余空间放大
  3. flex-shrink: 剩余空间不足，缩小
  4. flex-basic: 在分配空间之前占据的固定空间
  5. flex: flex-grow、flex-shrink和flex-basic 的简写
  6. align-self：该子元素的垂直方式
