webpack是什么

  • webpack将前端的资源文件(js/json/css/less/img…)都将作为模块处理，它根据模块的依赖关系进行静态分析，打包生成对应的静态资源

webpack核心模块1:entry

  • 入口：webpack打包从那个文件开始，创建内部依赖图

webpack核心模块2:output

  • 出口：打包生成的文件存放的地址

webpack核心模块3:loader（module）

  • laoder用于将源代码转换成可直接在浏览器运行的代码
  • 预处理文件
  • 将不通的语言转换成js，或者使用加载的样式文件、图片等
  • 可以允许配置多个loader，从下到上（从右到左）执行
    css-loader、less-loader、eslint-loader、postcss-loader、url-loader、babel-loader

webpack 核心模块4：plugins

  • plugins 的目的在于解决loader无法实现的事情
  • 使用的时候是向plugin创建一个new实例，可以传入参数

minicssextractplugin、htmlwebpackplugin

webpack 核心模块5：mode

  • 告知 webpack 使用相应环境的内置优化
  • 可能的值有：none, development 或 production（默认）

webpack 核心模块6：浏览器兼容

webpack性能优化

  • 开发环境
  • 生产环境

开发环境优化

  • 优化打包构建速度
  • 优化代码调试

生产环境优化

  • 优化打包速度
  • 优化运行速度

优化打包构建速度

  • 缓存 ：1.cacheDirectory 开启babel缓存 2. 静态资源缓存：hash：每次构建的时候都会生成一个hash值（重新打包会导致缓存失效）；chunkhash：根据chunk生成的hash值，如果打包来源来自同一个chunk，那么hash值就一样；contenthash：根据文件的内容生成hash值，不同的文件hash值不同
  • tree shaking（es6代码，production生产环境）: 去除无用代码
  • code split 1. 多入口文件 2. optimization :{splitChunks:{chunks:’all’}}
  • 懒加载、预加载 1. 使用import语法懒加载 2. prefetch 预加载，使用之前提前加载 ，等待其他资源加载完毕之后再去预先加载

devsrver模块配置

  • contentbase: 运行代码目录
  • compress: 启动gzip压缩
  • port：端口
  • host：localhost
  • open: 默认启动打开浏览器
  • hot:true 开启HMR功能
  • clientLogLevel: 日志级别
  • watchOption：忽略监听的文件
  • proxy：开发跨域代理 '/api':{target:'http://127.0.0.1:9000',}

optiization 模块配置

  • 必须要在生产环境下 （mode：production）

webpack 优化之HRM

  • 用于开发环境，在重新运行期间，替换、删除、修改文件的时候，无需重新加载整个程序

    webpack 优化之 tree shaking

  • 通常用于描述移除 JavaScript 上下文中的未引用代码(dead-code)
  • 开发环境下配置 optimization: {usedExports: true }
  • 生产环境下配置 "sideEffects": false(所有文件都是必须的) "sideEffects": [, "*.css"] (css 文件无需受到tree shaking 影响)
