vue3

模版语法

  1. v-once：一次性执行数据解析，当数据发生改变时候将不会重新更新数据
  2. v-bind：绑定标签属性使用动态数据 v-bind:herf 缩写为 :herf
  3. v-on: 绑定标签事件的 v-on:click 缩写为 @click

事件修饰符

  1. .stop:阻止单击事件继续冒泡
  2. .once: 事件只会触发一次
  3. .prevent:提交事件不再重载页面
  4. .self
  5. .passive: 配合 scroll 事件使用，滚动事件的默认行为将会立刻触发，不会等待 onscroll 完成
  6. .capture

按键修饰符

  • 你可以直接将 KeyboardEvent.key 暴露的任意有效按键名转换为 kebab-case 来作为修饰符。
  1. .enter:   监听 enter 键触发
  2. .page-down: pageDown 键触发
  3. tab
  4. delete
  5. esc
  6. space

  8. down

  10. right

全局注册组件

  1. 重新赋值
  2. 使用计算属性去修改

插槽

具名插槽

  • 在使用多个插槽的时候，在使用插槽的父组件中通过在标签上定义插槽名来渲染到对应的正确的组件插槽中
  • 使用： 1. 通过在slot 标签上通过 name 属性定义插槽名 2. 在父组件中通过在需要渲染到插槽的顶级标签上通过v-slot:[name]来选择需要渲染的标签

作用域插槽

  • 在普通插槽中，父子组件之间插槽使用数据渲染是相互不影响的，也就是：父组件的内容都是在父组件的作用域内编译的，子组件的内容都是在子组件的作用域内编译的
  • 为了解决在父组件中使用子组件插槽的内容，需要在父组件中使用子组件标签的时候通过 v-slot:default="scope"
    将子组件 slot 标签中绑定的数据传递到父组件中作为 scope 的属性；在子组件中通过slot标签上绑定数据``` ````

provide 和 Inject

  • 在父子或者爷孙组件之间可以通过 provide 来发送数据，在接受数据的组件中通过 Inject 来接受发送的数据
  • 注：在子组件之间不可进行数据传递，provide 和 Inject 的数据流向是通过 provide 在上层组件中向下流到下层组件通过 Inject 来接受

动态组件

  • 在 vue 中可以使用componment标签 搭配 is 属性来动态设置 componment 标签渲染的组件，is 的值为组件名

异步组件

  • vue 提供了使用 defineAsyncComponent函数来进行组件的异步加载

组合式 API

setup

  • 在 setup 函数中定义数据和方法，通过 return 返回出去，用于模版中直接使用
  • 参数(props 和 context)：props 参数用于存储组件的传值，context 参数中有 attrs:组件外部传递过来的，但没有在 props 中定义的属性；slots 属性：收到插槽的相关内容；emit：父子组件之间方法调用，与 vue2 中的 this.$emit 相似

ref 函数

  • 用来定义基本数据

reactive 函数

  • 用来定义引用数据

响应式原理

  • 通过 proxy 来拦截对象中的任意属性变化，使用 reflect 来操作源数据

多事件处理器

  • 事件处理程序可以有多个方法，这些方法将由逗号分隔

定义数据的方式

  • vue3 中通过ref 和reactive来定义数据

computed 计算属性

  • computed函数返回一个不可变的的响应式ref对象，
  • computed函数参数可以为一个get函数，或者 一个带有get和set的属性的对象

watch 数据监听

  • watch 函数接受 3 个参数：一个想要进行监听的响应式数据或者 getter 函数，监听回调函数，配置选项()

  • watch 也可以监听多个数据源,但是是以数组的形式,监听多个数据时，多个数据发生变化时也只会发生一次回调触发

  • 配置选项：
    deep:监听深度嵌套的对象或者数组变化，在监听深度嵌套的数据时还是需要对监听的深度嵌套的数据进行深度拷贝

  • 监听 ref 定义的数据 watch(count, (count, prevCount) => {})

  • 监听多个数据watch([fooRef, barRef], ([foo, bar], [prevFoo, prevBar]) => {})

  • 监听 reactive 定义数据的某个属性 watch(() => state.count,(count, prevCount) => {})

Teleport

  • 使用teleport标签 可以通过to属性来控制当前dom元素在那个父元素下进行渲染

watchEffect

  • watchEffect 传入一个函数，同时追踪其依赖，当依赖发生改变将会重新激活该函数

vue2 的响应式原理

  • vue2 通过使用Object.defineProperty get 和 set 方法来劫持数据监听数据变化，更新视图
  • 对象属性新增和删除不会引发视图更新、通过数组下标修改数据也不会引发视图更新
