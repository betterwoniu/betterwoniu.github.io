关于actived和deactivated的组件生命周期函数

只有该组件被<keep-alive>标签包裹才会触发actived和deactived两个钩子函数，表示该组件将会被缓存下来；
注意， 是用在其一个直属的子组件被开关的情形。如果你在其中有 v-for 则不会工作。如果有上述的多个条件性的子元素， 要求同时只有一个子元素被渲染。
include（只有名称匹配的组件会被缓存） 和 exclude（任何名称匹配的组件都不会被缓存） prop 允许组件有条件地缓存。二者都可以用逗号分隔字符串、正则表达式或一个数组来表示；
匹配首先检查组件自身的 name 选项，如果 name 选项不可用，则匹配它的局部注册名称 (父组件 components 选项的键值)。匿名组件不能被匹配。
max - 数字。最多可以缓存多少组件实例，最多可以缓存多少组件实例。一旦这个数字达到了，在新实例被创建之前，已缓存组件中最久没有被访问的实例会被销毁掉。

插槽

组件标签中的内容（可以是任何模版代码）将会在渲染的时候渲染在<slot> 的位置，如果不存在<slot>，将会抛弃组件标签中的内容

vue双向数据绑定原理

  1. 通过 Object.defineProperty 来实现：
  2. 1 在vue实例加载的时候，会先使用 Object.defineProperty来监听data的数据
  3. 2 监听数据完毕之后开始使用编译器处理页面中使用data中数据的地方，使用data数据的地方会创建一个观察者
  4. 3 当使用该数据的时候就会触发 Object.defineProperty中的get的回调方法，并将改观察者放入依赖收集器中，等待数据变化的时候触发
  5. 4 当数据变化的时候就会触发 Object.defineProperty中的set的回调方法，回调方法中存储的该数据的编译方法就会将新的值传进去，更新该数据的视图
