变量声明

  • let、const、var，let用于定义一个变量，不存在变量提升，使用let定义的变量，需要注意作用域；const用于定义一个常量，一旦声明赋值之后将不允许进行更改；需要注意作用域，在子作用域中可以重新声明定义，在该子作用域中使用的是该子作用域中定义的常量，同样也不存在变量提升；var定义的变量存在变量提升，提升定义不提升变量的值

    变量提升

  • 在声明定义变量之前使用该变量时，将会将该变量的声明提升，该变量为undefined，值不会提升

    暂时性死区

  • 在使用let 定义变量的时候，将会将该变量绑定到该作用域，在定义该变量之前使用该变量将会出现暂时性死区

优化一次性接受后端大批量数据展示的方式

情景描述：如果后端接口大批量数据列表展示，如何优化

  1. 使用settimeout 来根据每次展示的条数，进行多次渲染
  2. 使用requestAnimationFrame 来渲染，requestAnimationFrame是确保回调函数会把每一帧的dom操作都集中到下一次重绘和回流中完成，并且每次重绘和回流的时间都根据当前设备的最佳刷新频率来执行的(隐藏的dom元素将不会进行重绘和回流，页面不激活的时候将不会执行回调函数渲染)
  3. 文件碎片和requestAnimationonFrame结合优化，使用documment.createFrament 创建文件碎片，因为createFragment 创建的dom元素是存放在内存中的，向dom元素添加文件碎片的时候不回引起页面回流
  4. 懒加载：在渲染第一页数据的时候，根据列表结尾的空dom元素出现在视图中出现的时候进行下一次渲染

React

React 事件机制

  • react的事件监听不是绑定在真实的dom上的，而是通过document监听事件，当事件发生并冒泡到document的时候，react将事件内容封装转交给真正的处理函数进行。react是通过自己合成事件(syntheticEvent)，因此停止冒泡事件的方法应该是event.preventDefault()而不是event.stopProppagation()

jsx

  • jsx 是javascript的扩展，类似于模版语法，但是具有javascript的能力
  • jsx 是将dom转化为 react.createElement()生成的dom
  • babel将jsx的代码转化，再调用React.createElement 和ReactElement 创建虚拟dom ，再通过render方法转化为真实dom

React 和普通html 区别

  • 事件名称命名方式：html小写 react 小驼峰
  • 事件函数处理语法：html事件名为字符串，react 事件为函数
  • react只能通过preventDefult() 阻止事件默认行为
    避免原生事件和react的合成事件混合使用

React组件中如何做的事件代理

函数组件和类组件

  • 函数式组件是接收唯一的参数props并返回一个react元素，本质上是一个js函数；在函数组件中并不能够使用生命周期函数

  • 类式组件 通过继承 react.component 的类，该类中通过render方法返回一个react元素，在其构造函数中可以通过父类接收props、并直接使用state，可以在该类中直接使用生命周期函数

  • 函数组件对比类式组件来说性能更高，因为函数式组件只需要直接调用，而类式组件需要创建实例

React 生命周期函数

生命周期函数（旧）

  • 组件挂载时：constructor（组件创建）、componentWillMount(组件将要挂载)、render(组件渲染)、componentDidMount(组件已经挂载)
  • 组件更新时：shouldComponentUpdate(判断组件是否应该被更新，如果返回false，后续的render等渲染操作将不会被执行，如果返回true，则正常执行)、render、componentWillUpdate(组件将要更新)、componentDidUpdate（组件更新完毕）、componentWillReceiveprops(组件即将接受props的钩子函数)
  • 组件卸载时：componentWillUnmont（组建将要卸载)

    生命周期函数（新）

  • 组件挂载时：constructor、componentDidMount(组建挂载完毕)、render(组件渲染)、getDerivedStateFromprops(在Render函数执行前调用，参数接受props 删除了componentWillMount(组件将要挂载)钩子函数)
  • 组件更新时：shouldComponentUpdate(判断组件是否应该被更新，如果返回false，后续的render等渲染操作将不会被执行，如果返回true，则正常执行)、render、、getSnapshotBeforeUpdate(`最近一次的渲染前调用，删除了componentWillUpdate(组件将要更新)生命周期函数)、componentDidUpdate（组件更新完毕）、 删除了componentWillReceiveprops(组件即将接受props的钩子函数)
  • 组件卸载时：componentWillUnmont（组建将要卸载）
  • 新添加的钩子函数不会在第一次组件渲染的时候被调用

虚拟DOM和differ算法

  • virtual dom(虚拟dom)：virtual dom是一种编程概念，将dom以虚拟的形式存放在内存中，并通过ReactDom等库使之于真实的dom同步
  • 增量渲染：
  • diff算法：在页面遍历更新数据的时候，diff算法会将新生成的虚拟dom和更新前的虚拟dom进行比较，比较的标识是每个标签的index，如果index一样的话，就会比较标签中的内容，如果都一样的话，就不更新真实的dom，如果不一样的话，就将新生成的虚拟dom更新到真实的dom中。
  • 为什么不能用数组下标作为标识：如果使用数组下标作为每个dom的标识的话，在数组顺序发生改变的时候就会造成相同的标签需要重新进行渲染，如果使用数据的唯一标识作为index的话，不论数据发生怎样的改变都不会影响到原始真实dom的频繁更新。

  • 在创建store的时候，将reduce作为参数传递进去
  • reducer的本质就是一个函数，所以在创建reduce的时候是创建一个函数，reducer会接受两个参数（prestate（之前的状态），action（动作对象））
  • action中获取action和data

react-redux

  • react-redux 是通过创建一个容器组件包裹一个UI组件，组件之间通信通过props来传递
  • 容器组件是负责来操作store中的数据，而UI组件是用来做数据展示的
  • react-redux是通过自身的connect(mapStateToProps,mapDispatchToProps)(UI)方法来返回一个容器组件，mapStateToProps是用来传递UI组件中需要使用的的state数据;mapDispatchToProps是用来返回UI组件中操作数据的方法(mapStateToProps和mapDispatchToProps都是函数，返回UI组件所需要的数据和方法).UI是引入的UI组件
  • UI组件可以通过props来获取容器组件传递的a和b来展示或者操作数据的方法
  • react-redux中不需要再使用store.subscribe监听store数据变化
  • 借助provider来在根组件中传递store，不需要在每个容器组件中传递store

  2. 修改state中的数据，通过setState()传入一个函数来更新数据，该函数的参数时state和props

  • useEffect 第一个函数中返回回调函数，监听的是组件卸载的时候执行的动作 等价 componentWillUnmont函数

  • wacther 订阅者：订阅者用于observer监听器和模版解析compile之间的沟通，data数据发生改变，会通知订阅该数据的订阅者，订阅者则会触发模版解析器compile更新模版中的数据

vue如何实现数据绑定

  1. 通过 Object.defineProperty 来实现：
  2. 1 在vue实例加载的时候，会先使用 Object.defineProperty来监听data的数据
  3. 2 监听数据完毕之后开始使用编译器处理页面中使用data中数据的地方，使用data数据的地方会创建一个观察者
  4. 3 当使用该数据的时候就会触发 Object.defineProperty中的get的回调方法，并将改观察者放入依赖收集器中，等待数据变化的时候触发
  5. 4 当数据变化的时候就会触发 Object.defineProperty中的set的回调方法，回调方法中存储的该数据的编译方法就会将新的值传进去，更新该数据的视图

vuex

  • 集中式管理状态数据
  • 流程：1.组件中dispatch(派遣)action中的方法，action中commit(提交)mutations中的方法，mutations直接操作state中的数据; 2.组件中直接commit（提交）mutaitons中的方法，mutations直接操作state中的数据
  • vuex中定义mutations的方法时最好全大写方法名

vue 单向数据流

  • 在父子组件之间，props都是由父组件在使用子组件的时候传递过去的，数据是由父组件流向子组件，子组件不能直接修改props中的数据。父组建传递的props的数据发生改变，将及时更新子组件中的props的值。

  • 子组件修改需用使用emit通知父组件去修改props的数据，然后及时更新到子组件

computed和watch

  • computed计算属性需要依赖于其他的属性，当其他的属性改变的时候，下一次获取computed的值才发生改变
  • watch是监听它相关的数据，当相关数据发生改变，就会立即执行回调，更新数据

vue 中data 是一个函数

  • vue 中组件是可以复用的，每一个组件的data都是独立的，用函数返回是一个相互不影响的数据，而直接使对象的话，在js中对象属于引用类型，组件复用会造成数据污染

vue 组件之间通信

  • props和$emit：适用于父子组件之间通信
  • ref 、 $parent、$children: ref 获取指定DOM ，如果用于子组件，获取的就是子组件
  • EventBus:隔代组件
  • provide /inject：隔代组件
  • vuex

vuex

  • vuex 是vue框架用于进行状态管理的仓库
  • vuex 数据更新是响应式的，当数据发生改变，使用state的地方也同样发生改变
  • 修改vuex中的state状态，只能通过mutation提交修改
  • 模块：
    1. state：vuex中定义数据状态
    2. getter：获取state状态
    3. mutation：用于提交同步修改state的
    4. action：用于提交mutation，修改state，在action中进行异步操作
    5. module：将state按照业务拆分若干模块

vue 路由模式

  • hash模式：url路径中带有#，使用URL hash值来做路由，支持所有浏览器

         hash 模式的路由发生改变并不会向浏览器发送请求，只是通过hashchange方法监听路由变化，获取到#之后的路由，去router中寻找对应的组件重新渲染

  • history模式：依赖history Api和服务器配置

            history模式是通过 history.pushstate() 和history.replacestate()两个方法修改url，但是并不会去刷新页面。
        在修改url的时候就会触发popstate，在popstate中获取当前的路径，解析路径找到经营的组件重新渲染

  • abstract模式：支持所有javascript 环境
