javascript►CommonJS

export/import、require/exports

require 和 import是不同规范下引入模块的语句
require/exports 是2009年 CommonJs规范下引入模块的的方式
import/export 是2015年 es6 规范下引入模块的方式

类型 require/exports import/export
Node.js 所有版本 Node 9.0+（启动需加上 flag –experimental-modules Node 13.2+（直接启动）
Chrome 不支持 61+
Firefox 不支持 60+
Safari 不支持 10.1+
Edge 不支持 16+

CommonJs规范下的 require/exports 是针对于服务端开发的，服务器模块系统同步读取模块文件的内容，编译执行后得到模块接口（nodejs是CommonJs的实现）

在浏览器端因为异步加载的特性，commonJs无法正常加载。所以出现了RequireJs，SeaJs的模块化设计方案

  • import/export 在浏览器中是无法使用的，需要在加载的模块的<script>标签中使用 type="module"
  • 即使node 13.2+ 已经支持 import/export , Node官方不建议在正式环境使用，可以使用babel转化成CommonJs规范(具体实现还是require/exports)

require/exports 是运行时的动态加载，import/export 是静态编译

CommonJs加载的是一个对象（即module.exports 属性）, 该对象只有在脚本运行完才会生成，而es6模块不是对象，它的对外接口只是一种静态定义，代码静态解析阶段就会生成

  • es模块下 可以在import引用前使用，CommonJs则需要在require引用后使用

require/exports 输出的是一个值的拷贝，import/export输出的是一个值的引用

  • 改变输出的值对于require/exports的模块内部不会造成影响，对于 import/export 的内部会造成影响
  • import/export的模块不允许重新赋值/定义

exports 和 module.exports 的区别（commonjs）

  • 不可以以直接对exports直接赋值 （exports.hello = hello; function hello(){ console.log(hello)}）,使用module.exports 可以直接赋值
  • node 加载模块的时候 会将模块放入包装函数load中执行，在执行load函数前 将会定义一个变量
    var module = {
    id:’hello’,
    exports:{}
    }
    load 函数最终返回的是 exports；
    如果输出的是一个对象或者数组的话就要使用module.exports
    所以做好使用module.exports输出模块变量

    export 和 export defult 区别(es6)

  • exports defult 默认导出的数据
  • exports 是导出指定的数据

渲染►HTML

页面渲染

页面的渲染过程

  1. 将HTML构建成一个DOM树(DOM=Document object model 文档对象模型)，DOM的构建是一个深度遍历的过程：当前节点的所有子节点都构建好了才会去构建下一个兄弟节点
  2. 将CSS样式文件解析成 CSS Rule tree
  3. 根据DOM树和CSS树来构建一个 Rendering tree(渲染树)。注意：Render Tree渲染树并不等于 DOM树，因为像一些dispaly：none的标签就每必要渲染出来放在渲染树中
  4. 有了渲染树 Rendering Tree，浏览器能够知道页面的节点和节点的样式
  5. 页面根据 Rendering Tree 开始计算出每个节点的具体位置
  6. 开始绘制，遍历Rendering Tree 使用浏览器后端层绘制每个节点

回流(Reflow)和重绘(Repaint)

回流/重排(Reflow): 页面的某个部分发生了变化影响了布局，渲染树需要重新计算
重绘(Repaint): 改变了某个元素的背景颜色、文字颜色等，不影响页面的布局或者元素周围，只会引起浏览器的重绘(repaint)

如何减少回流(Reflow)

  1. 第一次的页面加载必定会触发依次回流/重排和重绘
  2. 触发重绘不一定会触发回流/重排，但是回流/重绘必定会触发重绘

触发回流的原因

  1. 元素在视图中的位置和大小发生改变
  2. 删除或者添加元素
  3. 浏览器的视图大小发生改变

CSS和JS的阻塞规则

  1. CSS不会阻塞DOM的解析，但会阻塞DOM渲染
  2. JS阻塞DOM解析，但是浏览器会“偷看”DOM，预先下载相关资源
  3. 浏览器遇到且没有defer或async属性的 标签时，可能会触发页面渲染，因此如果前面的CSS资源尚未加载完成，浏览器会等待完它加载完毕再执行脚本

PPI定义（物理像素）

PPI指的是每英寸上物理像素数数目，即“像素密度”。PPI的数值越大，屏幕效果越好。不过PPI是物理上的概念，是客观存在不会改变的值，跟开发中的DPI是完全不同。
物理像素又称为设备像素，是显示设备中的一个最小的的物理部件。每个像素可以根据操作系统设置自己的颜色和亮度

DPI 定义（设备独立像素）

DPI是指图像每英寸长度内的像素点数

显示网页内容的区域，可以通过js命令获取

layout viewport

网页布局的区域，它是html标签的父容器，很多时候浏览器窗口不能完全显示layout viewport的全貌，需要通过滚动条查看页面

ideal viewport

它与手机屏幕大小的相同，确切来说，等于visual viewport的大小。把页面渲染到这个ideal viewport里面，就能在visual viewport中完美显示

Promise

promise的含义

promise是一种异步编程的解决方案，promise是一个对象，保存着某个未来才会结束的异步操作事件的结果。promise的对象状态不受外界的影响，有三种状态：pending 、fulfilled 、rejected ，只有异步操作的结果可以决定是哪一种状态，任何其他的操作都不能改变其状态；一旦promise的状态发生改变就不会再变化。

promise.then()

then()方法是定义在原型对象 promise.protype上的，该方法的作用是状态改变的回调函数。then方法的第一个参数是resolved状态的回调函数，第二哥参数是rejected状态的回调函数。then方法返回的是一个新的promise实例，因此可以采用链式写法;链式结构的中then方法return的结果将作为参数传入给下一个then方法

Promise.prototype.catch()

catch()方法是用于指定发生错误的回调函数，当promise的状态改变为rejected的时候就不会调用then方法而是回调catch方法

Promise.prototype.finally()

finally() 方法用于指定不管promise对象最后状态如何都会执行的操作

Promise.all()

all()方法用于将多个promise实例包装成一个新的promise实例，返回一个数组存放各个promise的返回值。当多个primise的状态都变为filfilled，此时才返回值；当其中有一个被reject，新的promise的状态也会被reject

promise 缓存

场景1：将封装一个方法，将参数作为参数，每次调用的时候都进行异步请求查询

场景2：场景1可能会导致导致请求过于频繁，对于请求接口造成压力。所以，场景2采用缓存查询的promise来解决这问题

借助memoizee(https://www.npmjs.com/package/memoizee) 库来完善最后一个方案

单例模式Promise

async/await 函数

async 函数是解决异步方案，它是generator函数的语法糖，定义函数使用async关键字，await 命令后一般是一个promise对象，函数体中使用await 紧跟表达式需要等待的结果，并且async函数返回的是promise 对象。

async 函数返回一个promise 对象，函数内部返回的值，将成为 then方法中的参数。
async 函数内部抛出的错误，会使返回的promise对象的状态变为reject状态，抛出的错误会被catch方法捕捉到。
async 函数返回的是一个promise对象，必须要等到函数体内的 await 后的promise 对象执行完才会状态发生改变，或者return 语句或发生错误。只有async 函数体内中的异步操作执行完，才会执行then方法指定的回调函数。

async 错误处理机制：1.在函数体中使用 try…catch 捕捉错误；2.async函数使用catch方法捕捉错误

Generator 函数

generator函数是es6 提供的一种异步解决方案。
1.语法上generator函数是一个状态机，封装着多个内部状态
2.执行generator函数会返回一个遍历对象，所以generator函数也是一个遍历生成函数，返回的遍历对象可以依次遍历generator函数的每一个状态
3.generator函数定义的关键字是一个*，函数体内部使用yield表达式，遍历该函数对象通过该函数的next()方法，返回的是一个对象{ value: ‘hello’, done: false },当done的值为true时该遍历对象遍历结束。
4 任意一个对象的Symbol.iterator方法，等于该对象的遍历器生成函数，调用该函数会返回该对象的一个遍历器对象。

5.yield方法本身没有返回值，next方法可以带一个参数作为上一个yield表达式的值
6.for…of循环可以自动遍历generator函数运行生成的iterator对象，而不需要使用next方法
7.Generator.prototype.return()是可以给定参数，并终结该generator函数，如果return()方法不提供参数则返回value的值为undefined，如果generator函数体内存在try…finally，且正在执行try代码块，那么return方法就会进入finally代码块中，执行完毕，整个函数才会结束。
