@Input 和 @Output

@Input 和 @Output 是 angular 中父上下文（父组件？）和子组件、子指令共享数据，@Input 属性是可写的，数据流入；而@Output 是可观察的对象，允许向外发送数据

@Input 一般是在父上下文向子组件或者子指令传递信息 父组件的html页面中引入子组件或者子指令的时候 写入类似[pcmsg]="pmsg" pcmsg 是子组件或者子指令中定义接收父组件传递的数据变量，pmsg 是父上下文中定义要传递子组件或者子指令的变量

@Output 通常是将该属性初始化为 Angular EventEmitter,  并将值作为事件流出。 @Output() childmsg = new EventEmitter<string>(); this.childmsg.emit() 在子组件或者子指令中通过定义的事件流来向父上下文来传递数据

属性型指令

属性型指令是通过一个@Directive 装饰器的控制器的类获取到使用该指令的元素标签来控制该元素标签的操作。

结构型指令

结构型指令的职责就是HTML 布局，重塑DOM的结构 比如添加、移除或维护这些元素

组件交互

1.在父子组件中通过 @Input 和@Output 两个装饰器来交互

2.通过setter 截听输入属性值的变化

3.通过生命周期的钩子函数OnChanges 函数来监听数据变化

4.通过指令#XXX 来定义某组件，即可使用该组件中的函数和定义的属性变量

5.通过在父组件中使用@ViewChild装饰器来定义代表子组件的变量，即可使用子组件中的函数和定义的属性变量

angular 表单（模版驱动表单，响应式表单）

常用表单基础类

响应式表单和模版驱动表单都是建立下列基础类之上的：

  1. FromControl 实例用于追踪单个表单控件的值和验证状态
  2. FromGroup 用于追踪一个表单控件组合的状态和值
  3. FormArray 用于追踪表单控件的数组的值和状态
  4. ControlValueAccessor 用于angular的FromControl实例和原生DOM元素之间的一个桥梁

模版驱动表单

[FromControl]指令会通过内部值的访问器来把显示创建的FromControl实例与视图中的表单元素联系在一起

响应式表单

在模版驱动表单中，表单模式是隐式的，指令[(ngModel)]为指定的表单创建一个FromControl实例
